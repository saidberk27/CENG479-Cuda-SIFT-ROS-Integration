Proje Genel Bakış

Bu proje, iki görüntü arasında SIFT (Scale-Invariant Feature Transform) öznitelik çıkarımı ve FLANN (Fast Library for Approximate Nearest Neighbors) tabanlı eşleme yapan bir ROS2 pipeline'ının iki farklı versiyonunu içerir:

🐍

Sequential (Python)

İki ayrı ROS2 node'u (publisher + subscriber) ile çalışan, CPU tabanlı klasik pipeline. SIFT öznitelikleri ROS2 topic'leri üzerinden TCP socket ile iletilir.

⚡

Paralel (C++/CUDA)

Tek bir fused proses içinde, VRAM'de resident GPU pub/sub bus'ı kullanan yüksek performanslı pipeline. Sıfır kopyalama (zero-copy) ve GPU'da fused kNN + Lowe ratio test.

Temel Teknik Özellikler

  • ROS2 Humble altyapısı üzerinde çalışan modüler pipeline mimarisi
  • SIFT öznitelik çıkarımı (OpenCV implementation)
  • Sequential: FLANN KD-Tree tabanlı kNN eşleme + Lowe's ratio test
  • Paralel: GPU brute-force kNN + fused ratio test (CUDA kernel)
  • RANSAC homografi hesaplama ve perspektif dönüşüm
  • Detaylı performans metrikleri (wall/CPU time, stage bazında)
  • Benchmark sistemi ile iki implementasyonun karşılaştırılması

Veri Akışı (Data Flow)

Her iki pipeline'da da ortak olan adımlar (farklı implementasyonlarla):

  1. Görüntü Yükleme: large (ankara.jpg, 3228×2300) ve query (test.jpg, 775×775) görüntülerinin diskten okunması
  2. Grayscale Dönüşüm: BGR → gri tonlama (paralel versiyonda legacy CUDA kernel mevcut)
  3. SIFT Öznitelik Çıkarımı: ~123K (large) ve ~7.7K (query) keypoint
  4. Veri İletimi: Sequential'de ROS2 topic (TCP), Paralel'de VRAM bus (pointer hand-off)
  5. kNN Eşleme: Sequential'de FLANN KD-Tree (CPU), Paralel'de GPU brute-force + fused ratio test
  6. RANSAC Homografi: Her iki implementasyonda da OpenCV findHomography
  7. Perspektif Dönüşüm: Query köşelerinin large görüntüye izdüşümü

Sequential (Python/ROS2) Implementasyonu

⚠️ İki-Proses Mimarisi

Sequential pipeline iki ayrı ROS2 node'u olarak çalışır: sift_publisher ve flann_subscriber. Büyük deskriptör verisi (~62 MB) ROS2 topic'leri üzerinden DDS/TCP ile iletilir.

Publisher (sift_publisher.py)

Görüntüleri yükler, SIFT özniteliklerini çıkarır ve ROS2 ImageFeatures message'larına serileştirerek yayınlar. Her aşama detaylı performans metrikleriyle kaydedilir.

Stage 1: Image Loading

~61 ms'de iki görüntüyü diskten yükler.

Stage 2: Grayscale

~5 ms'de BGR→Gray dönüşümü. Ara görüntüler kaydedilir.

Stage 3-4: SIFT

Large: ~1655 ms (123K kp), Query: ~121 ms (7.7K kp). Seri çalışır.

Stage 5: Serialization

~808 ms'de ~62 MB + ~3.9 MB payload ROS2 message'ına dönüştürülür.

Stage 6: Publish

~105 ms'de iki topic'e yayın. İlk query, sonra large gönderilir.

Subscriber (flann_subscriber.py)

Publisher'dan gelen message'ları alır, deskriptörleri yeniden oluşturur, FLANN kNN eşleme yapar ve RANSAC homografi hesaplar. Görsel çıktılar matplotlib ile oluşturulur.

Transport Latency

Query: ~41 ms, Large: ~217 ms (DDS/TCP overhead)

FLANN kNN Matching

KD-Tree (trees=5, checks=50). ~1692 ms, ~4.5K matches/s

Ratio Test

Lowe's ratio (0.7). 7.7K → 79 good matches (%99 red)

RANSAC

~0.44 ms ile 79 noktadan 50 inlier (%63.3)

ROS2 Interface: ImageFeatures.msg

sift_interfaces/msg/ImageFeatures.msg — Görüntü adı, boyut, keypoint koordinatları (x, y, size, angle), düzleştirilmiş deskriptörler (n×128 float32) ve publish timestamp içerir.

Parallel (C++/CUDA) Implementasyonu

✅ Fused, Tek-Proses, CUDA-aware Pipeline

Publisher ve subscriber aynı proses içinde birleştirilmiştir. Veri VRAM'den hiç ayrılmaz; "yayınlama / abone olma" işlemi atomic pointer hand-off'udur. Socket ve ROS topic yükü tamamen kaldırılmıştır.

⚠️ SIFT Neden CPU'da? GPU'ya Taşınabilir mi?

Evet, SIFT extraction GPU'da yapılabilir. OpenCV'nin standart cv::SIFT::create() implementasyonu CPU tabanlıdır ve CUDA backend'i içermez. Ancak GPU'da SIFT çalıştırabilen üçüncü parti kütüphaneler mevcuttur:

  • SIFTGPU (Changchang Wu) — GPU'da SIFT'in en bilinen implementasyonu
  • PopSIFT (Cambridge) — OpenCL tabanlı, CUDA'ya da taşınabilir
  • NVIDIA CV-CUDA — CUDA-accelerated görüntü işleme kütüphanesi (SIFT feature extraction desteği)
  • cuCIM — NVIDIA'nın CUDA-accelerated görüntü işleme pipeline'ı

Bu projede OpenCV'nin standart CPU SIFT'i kullanılmıştır çünkü: üçüncü parti kütüphaneler ROS2/Catkin build sistemiyle entegrasyon zorluğu çıkarır, lisans farklılıkları olabilir ve SIFTGPU gibi kütüphaneler bakımı durdurulmuştur.

Mevcut SIFT paralelleştirmesi: İki thread'te eşzamanlı (large + query), her thread CPU'nun yarısını kullanır (cv::setNumThreads(half_cores)).

Temel Bileşenler

🚌

GpuFeatureBus

İki slotlu (query/large), mutex+condvar ile senkronize VRAM pub/sub kanalı. publish() atomic pointer hand-off, wait_for_pair() blokajlı bekleme.

🧠

GpuFeatures

Tamamen VRAM'de resident SIFT feature set handle'ı. Deskriptörler (N×128 float), keypoint bilgileri device pointer'larıyla tutulur. Custom deleter ile otomatik cudaFree.

⚡

CUDA kNN + Ratio Kernel

Tiled brute-force kNN (k=2, L2) + fused Lowe ratio test. Her warp bir query'yi işler, train tile'ları shared memory üzerinden stream edilir. SM'lerde maksimum verimlilik.

VRAM Bus Tasarımı

class GpuFeatureBus {
  enum class Slot { kQuery = 0, kLarge = 1 };

  // Publisher: atomic pointer hand-off (nanosaniye mertebesinde)
  void publish(Slot slot, shared_ptr<GpuFeatures> features) {
    lock_guard<mutex> lk(mtx_);
    slots_[static_cast<int>(slot)] = move(features);
    cv_.notify_all();
  }

  // Subscriber: her iki slot dolana kadar blokajlı bekleme
  pair<shared_ptr<GpuFeatures>,
        shared_ptr<GpuFeatures>> wait_for_pair() {
    unique_lock<mutex> lk(mtx_);
    cv_.wait(lk, [this] {
      return slots_[0] != nullptr && slots_[1] != nullptr;
    });
    return {slots_[0], slots_[1]};
  }

  shared_ptr<GpuFeatures> slots_[2];
  mutex mtx_;
  condition_variable cv_;
};

CUDA Kernel Mimarisi

kernel_knn_match_ratio — Tiled Brute-Force kNN + Fused Ratio Test

🔬 Kernel Tasarım Detayları

// Her warp bir query descriptor'ı işler.
// Train descriptor'ları shared memory'ye TILE_T=64 satırlık tile'lar halinde
// yüklenir ve QUERIES_PER_BLOCK=8 warp tarafından ortaklaşa kullanılır.
// Bu, global memory trafiğini O(Nq×Nt) → O((Nq/8)×Nt) azaltır.

constexpr int WARP             = 32;    // thread per warp
constexpr int QUERIES_PER_BLOCK = 8;    // warps per block
constexpr int TILE_T            = 64;   // train rows cached per tile

__global__ void kernel_knn_match_ratio(
    const float * __restrict__ d_query,   // Nq × 128
    const float * __restrict__ d_train,   // Nt × 128
    int Nq, int Nt,
    float ratio_sq,
    int * __restrict__ d_train_idx,
    unsigned char * __restrict__ d_pass)
{
    // Shared tile: TILE_T × 128 train descriptors + QUERIES_PER_BLOCK × 128 query
    __shared__ float4 s_train[TILE_T][32];      // 128/4 = 32 float4
    __shared__ float4 s_q[QUERIES_PER_BLOCK][32];

    // Her warp query'sini shared memory'ye yükler
    // (her lane kendi float4'ünü __ldg ile getirir)

    // Ana döngü: train tile'ları üzerinde stride
    for (int t0 = 0; t0 < Nt; t0 += TILE_T) {
        // Cooperatif shared memory yükleme (coalesced)
        // __syncthreads() ile senkronizasyon

        // Her train satırı için 128-dim L2 mesafe hesabı
        // Warp reduction: __shfl_down_sync ile 32 lane → full distance
        // Her lane 1 float4 (4 eleman) işler
    }

    // Çıktı: Lowe ratio test (squared form, sqrt'siz)
    //   keep  iff  best < ratio² × second
}

Kernel Performans Özellikleri

Parametre Değer Açıklama
Block size (32, 8) 1 warp × 8 warps = 256 thread/block
Grid size ceil(Nq / 8) 7.7K query → 962 blocks
Shared memory (64+8)×32×16 = 36.8 KB float4 × 72 × 32 = 9216 bytes → çok düşük
Query descriptor 128 floats = 32 float4 1 warp lane = 1 float4 (4 eleman)
Train tile 64 satır × 128 eleman Tüm warplar ortak kullanır
Mesafe hesaplama L2 squared (sqrt'siz) 32 lane → __shfl_down_sync reduction
Ratio test Fused (ayrı kernel gerekmez) 0.7² = 0.49 threshold

Neden Tiled Yaklaşım?

SIFT eşleme asimetrik bir problemdir: 7.7K query × 123K train. Her query tüm train seti ile karşılaştırılır. Tiled yaklaşım ile train descriptor'ları shared memory'ye bir kere yüklenir ve 8 farklı warp tarafından tekrar kullanılır. Bu, global memory okuma trafiğini ~8× azaltır.

📊 Bellek Trafiği Hesabı

Tiled olmadan: 7.7K × 123K × 128 × 4 bayt = 485 GB global okuma
Tiled ile: (7.7K/8) × 123K × 128 × 4 bayt = 60.6 GB (~8× azalma)

Veri Yolu (Pipeline)

CUDA-aware Zero-copy Path:

CPU SIFT → cudaMemcpyAsync (H2D, pinned) → VRAM (d_descriptors) → GpuFeatureBus (pointer hand-off) → kernel_knn_match_ratio (GPU) → D2H (tiny match indices) → CPU RANSAC

✅ Hiçbir deskriptör CPU'ya geri dönmez. Sadece eşleşme indisleri (birkaç KB) D2H transfer edilir.
